SceneDetection Computer Vision Service

Available Services

Service Name : Object Detection
Endpoint : /start-object-detection/<video_id>/<progress_endpoint>
Arguments :
video_id : _id field of the document of the video in videos collection in database, that is going to be used in detection.
progress_endpoint : An optional argument, which will be used in HTTP GET request to send progress information. This request will be sent to an endpoint which is a combination of progress_endpoint, progress information (percentage) and video_id which looks like: <progress_endpoint>/<video_id>/<progress>. This argument can be used for informing clients about the progress in real time.
Response Fields :
status : OK if no errors occured, otherwise ERROR. Runtime erros won't be caught with HTTP request - response based architecture services, so, if any error occurs in runtime, their logs will be inserted into logs collection.
operation_id : An ID which can be later used to terminate detection.
Description : With this service, objects can be detected in videos. With supplied arguments, objects will be detected and for every frame, detected objects will be inserted into detected_objects collection in database. Inserted documents have a shape which looks like: {"class": int, "confidence":float, "x":float, "y":float, "w":float, "h":float, "frame_no":int, "video_id":ObjectId}. Also, service will update object_detection_status and object_detection_progress fields of the video document while detecting objects. When detection is completed , object_detection_status field will be updated to COMPLETED. While detection occurs, object_detection_progress field will be updated with calculated progress percentage.
Service Name : Termination Object Detection
Endpoint : /terminate-object-detection/<operation_id>
Arguments :
operation_id : This ID can be found in the response of Object Detection service.
Response Fields :
status : OK if no errors occured, otherwise ERROR. Runtime erros won't be caught with HTTP request - response based architecture services, so, if any error occurs in runtime, their logs will be inserted into logs collection.
Description : Detection operation with ID operation_id will be terminated. object_detection_status will be updated to ERROR. object_detection_progress field won't be updated. Changes made to detected_objects collection will be reverted.
